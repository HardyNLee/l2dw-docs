# 文件格式

此页面介绍用户在使用过程中，可能会碰到的一些文件格式。

## conf 和 wmdl

L2DW 专用的「立绘配置」文件格式，它并不等同于 Live2D 的 Json 文件。该文件描述了如何导入 Live2D 模型，并且记录 WebGAL 脚本模板，如何组合子立绘等信息。

- `.conf` 是 L2DW 旧版立绘配置文件格式。
- `.wmdl` 是 L2DW 新版立绘配置文件格式（从 1.4.24 起）。

L2DW 支持导入以上两种格式，导出的时候会统一导出 `.wmdl` 格式。

推荐使用 `.wmdl` 格式，其优势在于：

- 方便分享给他人使用。

  旧版 `conf` 记录的是机主本人的文件路径，分享给他人时该路径大概率失效，导致无法正确导入立绘。而新版 `wmdl` 记录的是相对路径，分享给他人时只要保持文件夹结构不变即可正确导入。

- 直接在 MyGO 3.1.0 及以上版本中使用。

  MyGO 3.1.0 及以上版本原生支持导入 L2DW 的 `.wmdl` 立绘配置文件，导入后即可直接使用，作为「拼好模」带来了极大的便利。

- 获取未来的功能支持。

  L2DW 未来的新功能可能只会支持新版 `.wmdl` 格式，旧版 `.conf` 格式可能会逐渐被淘汰。

## json

> 注意和 `.exp.json` 区分开来。

Live2D 旧版模型的描述文件格式，L2DW 仅支持导入该格式的文件。该文件记录了 Live2D 模型的各项信息。

如果您的模型是 `.model3.json` 格式（Live2D 新版模型），L2DW 暂不支持导入该格式的模型。

## png 和 jpg

L2DW 目前支持的图片格式。

## wmot

L2DW 专用的动作文件格式，记录了关键帧位置，曲线，时长，帧率等信息。

> 若需要给 Live2D 模型作为动作使用，请导出为 `mtn` 格式。

## mtn

旧版 Live2D 模型的动作文件格式，可被 Live2D 模型直接使用。

> `mtn` 文件记录的是烘焙后的关键帧数据，损失了很多有效信息，不便于再次编辑。若需要保留有效信息，请使用 L2DW 专用的 `wmot` 格式。

## exp.json

形如 `表情名称.exp.json` 的文件，是旧版 Live2D 模型的表情描述文件格式。
